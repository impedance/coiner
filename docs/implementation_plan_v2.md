# План реализации Coiner (Moneywork) MVP v2

Этот документ описывает шаги по превращению текущего прототипа в полноценную систему управления финансовым поведением с акцентом на осознанность (awareness) и премиальный пользовательский опыт.

## Цели и Принципы (v2)
1. **Поведение выше учета**: Приложение должно помогать *формировать привычки*, а не просто фиксировать траты.
2. **Премиальная эстетика**: Интерфейс должен вдохновлять, быть визуально "дорогим" и отзывчивым (Glassmorphism, HSL-цвета).
3. **Осознанное распределение**: Каждому рублю — своя задача («heap distribution»).

---

## Фаза 1: Архитектура и Структура Категорий
*Цель: Перейти от плоского списка к логическим группам (фиксированные, переменные, накопления).*

1. **База данных**:
   - [ ] Создать таблицу `category_groups` (id, name, sort_order).
   - [ ] Добавить `group_id` в таблицу `categories`.
   - [ ] Мигрировать системные категории в соответствующие группы.
2. **UI (Plan Screen)**:
   - [ ] Группировка категорий в списке.
   - [ ] Отображение прогресса ("Spent/Available") на уровне группы.
   - [ ] Визуальное выделение системных бакетов (**Reserve**, **Joy Fund**).

## Фаза 2: Дизайн-система и Визуальный "WOW-эффект"
*Цель: Сделать приложение эстетически превосходным.*

1. **Стилизация**:
   - [ ] Внедрение единой палитры HSL (vibrant colors для доходов, спокойные для расходов).
   - [ ] Glassmorphism-эффекты для карточек на Dashboard.
   - [ ] Современная типографика (загрузка шрифтов Inter/Outfit).
2. **Микро-анимации**:
   - [ ] Анимированные прогресс-бары накоплений.
   - [ ] Feedback-анимация при успешном сохранении транзакции ("Haptic success").

## Фаза 3: Мастер Распределения Доходов (Income Wizard)
*Цель: Реализовать метафору «раскладывания денег по кучкам».*

1. **Функционал**:
   - [ ] Создать отдельный flow для распределения только что полученного дохода.
   - [ ] Автоматическое отчисление 10% в "Reserve" (опционально, настраиваемо).
   - [ ] UI-индикатор: "Сколько осталось распределить из текущей суммы".
2. **Интеграция**:
   - [ ] Улучшить переход с экрана "Add Income" в мастер распределения.

## Фаза 4: Поведенческие Ритуалы и Циклы
*Цель: Усилить психологическую значимость действий.*

1. **Weekly Review (Ритуал)**:
   - [ ] Переработка экрана обзора: добавить графики динамики резерва.
   - [ ] Добавить секцию «Победы недели» (Celebrations).
2. **Practices (Привычки)**:
   - [ ] Улучшить визуализацию стриков (streaks) на Dashboard.
   - [ ] Добавить уведомления-напоминания о ежедневных практиках.

## Фаза 5: Портативность и Финализация
1. **Data layer**:
   - [ ] Реализовать экспорт/импорт JSON в файлы устройства.
   - [ ] Добавить функцию "Reset local data" с подтверждением.
2. **Верификация**:
   - [ ] Проведение Smoke-тестов (structural + functional).
   - [ ] Проверка соответствия RFC-001 по всем пунктам.
